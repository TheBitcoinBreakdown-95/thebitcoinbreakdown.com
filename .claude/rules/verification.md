# Verification Before Done

- Never mark work complete without demonstrating correctness
- Validate output against the original request -- did it actually solve the problem?
- Run the project's build/test/lint command to verify nothing is broken
- Use binary pass/fail criteria -- "it looks right" is not verification
- If the same approach fails twice, stop and re-evaluate -- do not loop
- Prefer proof (test output, build log, screenshot) over explanation
- Ask: "Would a senior developer approve this?"
