## Part 2: Source Inventory

### Core Properties (Author's Outline)

1. Bitcoin is open source -- the code is fully auditable and can be copied and distributed for free, like stock images. (Author)
2. Bitcoin operates through open collaboration. (Author)
3. Bitcoin is accountable -- every transaction ever is free to see by anyone. (Author)
4. Everyone validates in the Bitcoin network. (Author)
5. There is a trade-off between an open monetary policy and privacy. (Author)
6. Bitcoin is extremely transparent -- its code is completely open source, meaning anyone can read or alter it. (Author)
7. All transactions are broadcast to everyone, and you can read every transaction since the genesis block. (Author)

### Bitcoin Not a Scam (Transparency vs. Fraud)

8. Absolutely nothing about Bitcoin is a secret -- it's open source, anyone can review the code, anyone can contribute, anyone can run the software voluntarily, and anyone can use the network without permission. (Author)
9. The entire history of all Bitcoin transactions is visible to anyone in the world. (Author)
10. This is the total opposite of a fraudulent investing scam, which is shrouded in vague promises of high returns with capital inflows and outflows kept in a secret ledger. (Author)

### Reddit FAQ

11. Bitcoin code is fully auditable -- you can read and contribute to the source code yourself. (Reddit r/Bitcoin FAQ)
12. The public ledger is transparent -- all transactions are seen by everyone. (Reddit r/Bitcoin FAQ)

### Open Source Philosophy

13. Open source software empowers individuals around the world while remaining independent of governments and large corporations that continuously attempt to track, censor, and control. (b.tc/conference/opensource)
14. Many open source projects are not run for profit and do not have large budgets -- they rely on an ever-increasing network of individuals worldwide who contribute time and energy, often without compensation. (b.tc/conference/opensource)

### Open Collaboration / BIP Process

15. Any attempts to make protocol changes, especially non-backwards-compatible changes, should occur in the open rather than behind closed doors. (nakamoto.com)
16. Bitcoin belongs to humanity -- proposed changes must be open to public comment. (nakamoto.com)
17. The Bitcoin Improvement Proposal process is the recommended way to suggest changes, though no authority can enforce that the process be followed. (nakamoto.com)
18. The issue of voluntary organization and the power dynamics that result can create the perception that specific people or groups are authorities -- but this is an illusion of power. (nakamoto.com, citing Lopp)
