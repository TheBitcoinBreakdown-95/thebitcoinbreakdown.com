## Part 2: Source Inventory

### Core Property

1. Bitcoin is peer-to-peer -- no intermediaries taking a cut, no need for trusted third parties. (Reddit r/Bitcoin FAQ)

### Network Architecture Definitions (Wikipedia)

2. Client-server architectures: smart clients contact the server for data, then format and display it; input at the client is committed back to the server when it represents a permanent change. (Wikipedia)
3. Three-tier architectures: move client intelligence to a middle tier so that stateless clients can be used, simplifying application deployment; most web applications are three-tier. (Wikipedia)
4. n-tier architectures: web applications that further forward requests to other enterprise services; most responsible for the success of application servers. (Wikipedia)
5. Peer-to-peer architectures: no special machines provide a service or manage network resources; all responsibilities are uniformly divided among all machines known as peers; peers can serve as both clients and servers. (Wikipedia)
6. Examples of peer-to-peer architecture include BitTorrent and the Bitcoin network. (Wikipedia)

### Satoshi Quote on P2P Resilience

7. "Governments are good at cutting off the heads of a centrally controlled network like Napster, but pure P2P networks like Gnutella and Tor seem to be holding their own." (Satoshi Nakamoto)

### Reliability

8. A peer-to-peer network can provide more reliability than a non-distributed system because there is no single point of failure. (Wikipedia)
